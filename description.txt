php.ini
	- по умолчанию настроен так чтобы не включать данные файлов в cookie в массиве $_REQUEST

closeCursor();
	- освобождает соединение с сервером давая возможность запускать другое SQL соединение

$_POST
	- получаем все переменные формы из метода ПОСТ массивом
$_GET
	- из метода ГЕТ массивом (прописывает указаное значение в строку УРЛ при отправке)

PHP_SELF 
	- выводит значение УРЛ пути из того места откуда вызванна 
$_REQUEST
	- её ключи создаються всеми значениями ? сгерерированными в текущем HTTP-запросе браузером пользователя.
		Это означает что она может получить доступ ко всем данными, хранящимся в переменных $_GET и $_POST,
		кроме того любые данные хранящиеся в файлах cookie браузера также храняться в переменной $_REQUEST.
		- по возможности Удалять эту переменную из ПХП кода 
$_SERVER 
	- в него PHP-интерпретатор помещает переменные, полученные от сервера.
 
htmlspecialchars() 
	- удаляет из URL-адруса нежелательные символы что позволяет предотвратить атаки XSS

strip_tags() 
	- удаляет из сообщения любые ХТМЛ-теги

explode() 
	- создаёт новый массив и ключ каждый раз когда функция обнаруживает пробел в тексте содержимого.

array_slice()
	- используется для отбора первых дясити ключей в нашем массиве созданном функцией explode();
